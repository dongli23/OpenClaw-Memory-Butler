# 🦞 龙虾内存管家 (Lobster Memory Butler) v1.2
> **拒绝 API 报错卡死，让你的 OpenClaw 拥有“痛觉”与“自愈”能力**

## 🌟 为什么需要它？
很多“小白”用户在使用 OpenClaw（“火山龙虾”）时，常会遇到聊着聊着突然卡死、系统报错不再回复的情况。

**崩溃真相：**
* **框架误导**：很多框架默认设置的上下文容量（如 200,000 tokens）远超了大模型 API 的真实物理上限。
* **物理天花板**：以 DeepSeek-V3.2 为例，其物理极限是 **128,000 tokens**。
* **逻辑冲突**：当对话量穿透 128k 的天花板时，API 会直接报错，而此时框架（如设定 70% 即 140k 触发压缩）还没来得及反应，导致系统直接崩掉。

**龙虾内存管家** 正是为了终结这一痛点而生的“避坑神器”。

---

## ✨ 核心功能
* **物理容量自动校准**：内置主流模型物理容量库，自动将“虚假参数”修正为真实的 API 物理红线。
* **智能双模反馈**：
    * **全貌模式 (Comprehensive)**：展示详细的内存报表、风险等级和操作建议，带你读懂大脑状态。
    * **精简模式 (Routine)**：日常使用仅输出一行极简状态，极致节省你的 Token 消耗。
* **人设自适应预警**：当占用率达到 60% 或 85% 时，会根据你设定的人物口吻自然地提醒你“脑容量快装不下了”。
* **小白友好指令**：无需背诵英文，直接回复“精简记忆”、“清空大脑”或“查看全况”即可。

## 🛠️ 安装指南

### 1. 配置 Skill 技能 (JSON)
在 OpenClaw 的 Skills 管理后台添加如下定义：
```json
{
  "name": "lobster_memory_management",
  "description": "龙虾内存管家。支持 compact(压缩)、reset(重置)、status(精简状态)、full_status(全貌报告)。",
  "parameters": {
    "type": "object",
    "properties": {
      "action": { "type": "string", "enum": ["compact", "reset", "status", "full_status"] }
    },
    "required": ["action"]
  }
}
